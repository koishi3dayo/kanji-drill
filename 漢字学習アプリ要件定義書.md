# 漢字学習アプリ 要件定義書

- **プロジェクト名**：漢字ドリル（小学生向け 漢字学習アプリ）
- **作成日**：2026-07-03
- **バージョン**：v2.8（予習／復習モード・習った漢字リスト：実装完了）
- **開発手法**：AI-DLC（AI-Driven Life Cycle）

---

## 1. 背景・目的

小学2〜5年生（特に4〜5年生重点）の漢字定着を、**毎日10問のドリル**で習慣化する。
学習の継続と達成感を、進捗の可視化（ヒートマップ）とゲーミフィケーション（連続日数・バッジ）で支える。
**（v2.7）対象範囲を小学6年生・中学1年生まで拡張し、出題数（10問／20問）と出題順序（ランダム／五十音順）を選べるようにする。**

## 2. 仕様変更の記録

### 2.1 変更の経緯

| 版 | 入力・採点方式 | 課題 |
|----|--------------|------|
| v1.0（当初） | PCのキーボード＋IMEで漢字を入力して自動採点 | IMEが漢字を候補表示するため「思い出して書く」練習にならない。学習効果が薄い |
| v1.5（検討） | タブレット/スマホで手書きキャンバス入力 | 手書き対応端末が必要。本PCは非対応 |
| **v2.0（本仕様・確定）** | **PC出題 → 紙に手書き回答 → PCで答え合わせ・自己採点** | 課題を解消。本物の書き練習は紙で行う |

### 2.2 v2.0 の基本方針（確定仕様）

1. **電子端末への手書き入力は行わない。**
2. **PCが問題を出題する。**（読み・熟語・用例のヒントを画面表示）
3. **児童は問題を見て、紙に漢字を手書きで回答する。**
4. **PCで正解を表示し、児童本人または保護者が答え合わせをする。**
5. **正解／不正解をPC上でチェックボックス（○×）で記録する。**
6. **記録の履歴から、学習進捗と弱点をヒートマップで可視化する。**

## 3.5 プロフィール管理（v2.4 新規・実装完了）

### 3.5.1 背景
1家庭に子どもが3人（1号・2号・3号）おり、学年がそれぞれ異なる。1台の端末を共用するが、**進捗・実績カレンダー・出題の重み付けは子どもごとに独立して管理する**必要がある。

### 3.5.2 仕様（確定事項）
- **プロフィールは「1号」「2号」「3号」の3人固定。**（4人目以降の追加や動的な人数変更は本仕様では対象外）
- 各プロフィールは以下を個別に持つ：
  - **学年（2〜6年・中学1〜3年から選択）**：出題の重み付けの基準になる（§4.1.1）。**（v2.7）6年生・中学1〜3年を選択肢に追加**
  - **出題数（10問／20問、v2.7新規）**：1回のテストで出す問題数
  - **出題順序（ランダム／五十音順、v2.7新規）**：問題を並べる順序
  - **学習モード（予習／復習、v2.8新規）**：出題の元になる漢字プールの選び方（§4.1.4）
  - **習った漢字リスト（v2.8新規）**：学校で習った漢字の記録。復習モードの出題元になる（§4.1.5）
  - **学習履歴**（実施日・出題内容・正誤）→ 学習カレンダーに反映
  - **漢字ごとの正答統計**（出題回数・正解数）→ しんちょくマップ（ヒートマップ）に反映
  - **連続日数（ストリーク）・バッジ**
- **プロフィール切り替えUI**をホーム画面の最上部に設置する（例：`[1号] [2号] [3号]` の3択タブ）。
  - 選んだプロフィールが「現在アクティブな子ども」となり、以降の操作（学年選択・今日の10問・進捗確認・カレンダー確認）はすべてそのプロフィールに対して行われる。
  - これにより「今日の10問を始める、と同じ画面で誰がやるか選ぶ」「トップ画面で誰が使うか選ぶ」という2つの要望を、**1つのプロフィール切り替えUI**で同時に満たす（同じ選択が両方の目的を兼ねる）。
- **学年選択UI**を、プロフィール切り替えのすぐ下に設置する（例：`[2年] [3年] [4年] [5年] [6年] [中1] [中2] [中3]` タブ、現在の選択がハイライト）。
  - アクティブなプロフィールの学年を表示・変更する。変更すると即座にそのプロフィールの出題重み付け（§4.1.1）に反映される。
  - 学年は年に数回程度しか変わらない想定のため、**「毎回選び直す」のではなく「プロフィールに保存された設定を、必要な時だけ変更する」形**にする。
- **（v2.7）出題数選択UI・出題順序選択UIを、学年選択のすぐ下に追加する。**
  - 出題数：`[10問] [20問]` の2択タブ。
  - 出題順序：`[ランダム] [五十音順]` の2択タブ。
  - いずれもプロフィールに保存され、変更した回から反映される。
- **（v2.8新規）学習モード選択UIを、出題数・出題順序と同じ並びに追加する。**
  - `[予習モード] [復習モード]` の2択タブ。
  - **予習モード**：これまでどおりの動作（§4.1.1の学年重み付け）。**変更なし。**
  - **復習モード**：習った漢字リスト（§4.1.5）から出題する（§4.1.4）。
- **（v2.8新規）「習った漢字を選ぶ」ボタンをホーム画面に追加する。**
  - タップすると、**現在の学年の漢字だけ**を一覧表示し、学校で習った漢字をチェック（複数選択）できる画面に遷移する。
  - 一つ下以下の学年は選択操作が不要（自動的に全て「習った」扱いになるため、§4.1.5）。
  - 初期状態は**全て未選択**。保護者・本人がまとめてチェックを入れていく。
  - 選択件数（例：「132/191字 選択ずみ」）を画面に表示し、選び忘れを防ぐ。
- ホーム画面のストリーク・週間成績・バッジ・「今日の10問を始める」ボタンは、**すべてアクティブなプロフィールのデータを表示する。**（v2.7以降は出題数に応じて「今日の◯問を始める」と表示）
- **しんちょくマップ（ヒートマップ）・学習カレンダーも、アクティブなプロフィールのデータのみを表示する。**（画面上部に「1号の記録」等、現在どのプロフィールを見ているか分かる表示を出す）
- **漢字いちらん画面（漢字辞書としての一覧）は共通データのため、プロフィールに依存せず全員共通のまま**とする（漢字そのものの読み・熟語は学習進捗ではないため）。

### 3.5.3 データ保存構造（設計案）
現行の単一構造から、プロフィールごとに分離した構造へ変更する。

```jsonc
{
  "activeProfile": "1",              // 現在アクティブな子ども ("1"|"2"|"3")
  "profiles": {
    "1": {
      "name": "1号", "grade": "5", "qCount": 10, "orderMode": "random",
      "studyMode": "preview",          // "preview"(予習) | "review"(復習)
      "learnedByGrade": { "5": ["歌","算","給", "..."] },  // 現在の学年ぶんの明示選択（下の学年は自動で全て習った扱い）
      "history": {...}, "stats": {...}, "streak": {...}
    },
    "2": { "name": "2号", "grade": "3", "qCount": 10, "orderMode": "random", "studyMode": "preview", "learnedByGrade": {}, "history": {...}, "stats": {...}, "streak": {...} },
    "3": { "name": "3号", "grade": "2", "qCount": 10, "orderMode": "random", "studyMode": "preview", "learnedByGrade": {}, "history": {...}, "stats": {...}, "streak": {...} }
  }
}
```
- **（v2.7新規）`qCount`**（10 or 20）・**`orderMode`**（"random" or "gojuon"）をプロフィールに追加。未設定時は `qCount:10`・`orderMode:"random"`（従来どおりの挙動）にフォールバックし、既存データへの後方互換を保つ。
- **（v2.8新規）`studyMode`**（"preview" or "review"）・**`learnedByGrade`**（学年コード→習った漢字の配列、のマップ）をプロフィールに追加。未設定時は `studyMode:"preview"`・`learnedByGrade:{}`（＝下の学年のみ習った扱い）にフォールバックし、既存データへの後方互換を保つ。
  - `learnedByGrade` は**現在選択中の学年のキーだけ**を読む（下の学年は常に自動で全て含まれるため保存不要）。学年を変更しても、以前の学年で選んだ内容はキーごと保持され、その学年に戻したときに復元される。
- `history` / `stats` / `streak` の中身は既存仕様（§5.2・§6）と同じ形式を、プロフィール単位で持つ。
- **移行措置**：旧バージョンの単一データが端末に残っている場合、初回起動時にそれを「1号」のデータとして自動移行し、消失させない。

## 4. 出題仕様

### 4.1 出題数・頻度
- **1日1回のセッションを基本とする**（再挑戦は可）。
- **10問すべてを最初から画面に一覧表示する。**1問ずつの逐次出題は行わない。
- **（v2.7新規）出題数はプロフィールごとに「10問」「20問」から選択できる。**（§3.5.2、§4.1.2）
- **（v2.7新規）出題順序はプロフィールごとに「ランダム」「五十音順」から選択できる。**（§4.1.3）

### 4.1.1 出題学年の重み付け（v2.4 改訂：プロフィールの学年に応じて可変／v2.7で6年・中学1〜3年に対応）

v2.2では「5年生固定・9:1」だったが、v2.4では**アクティブなプロフィールの学年設定（§3.5）に応じて重み付けが変わる。**v2.7では対象学年に6年生・中学1〜3年を追加し、学年の並び順は
`2年 → 3年 → 4年 → 5年 → 6年 → 中学1年 → 中学2年 → 中学3年` とする（各学年の「一つ下」は配列上の前の学年。例：中学2年の一つ下＝中学1年）。

| プロフィールの学年 | 出題比率（10問設定の場合） |
|---|---|
| 2年生 | 2年生の漢字が **10問中10問**（2年生より下の学年がないため全問2年生） |
| 3年生 | 3年生の漢字が **10問中9問**、残り1問は主に2年生（一つ下の学年） |
| 4年生 | 4年生の漢字が **10問中9問**、残り1問は主に3年生（一つ下の学年） |
| 5年生 | 5年生の漢字が **10問中9問**、残り1問は主に4年生（一つ下の学年） |
| **6年生（新規）** | 6年生の漢字が **10問中9問**、残り1問は主に5年生（一つ下の学年） |
| **中学1年（新規）** | 中学1年の漢字が **10問中9問**、残り1問は主に6年生（一つ下の学年） |
| **中学2年（新規）** | 中学2年の漢字が **10問中9問**、残り1問は主に中学1年（一つ下の学年） |
| **中学3年（新規）** | 中学3年の漢字が **10問中9問**、残り1問は主に中学2年（一つ下の学年） |

- 「主に」としているのは、**前日の不正解の復習を優先する際、その復習問題が必ずしも一つ下の学年とは限らないため**（例：5年生プロフィールで3年生の問題を前日間違えていた場合、その復習を1問枠に充てる）。復習対象がない場合は、一つ下の学年から正解率の低い漢字・未出題の漢字を優先して選ぶ。
- 前日不正解の復習を優先する際も、上記の学年比率（例：5年生プロフィールなら9:1）は維持する。
- **20問設定の場合は、上記の比率を20問ベースに換算する**（例：5年生なら18問＋一つ下の学年2問。§4.1.2）。

### 4.1.2 出題数の選択（v2.7 新規・実装完了）
- プロフィールごとに **10問／20問** を選択できる（§3.5.2）。
- 学年重み付け（§4.1.1）の比率はそのまま維持し、問題数に応じて比例配分する。
  - 10問：主学年9問／一つ下1問（2年生のみ全問主学年）
  - 20問：主学年18問／一つ下2問（2年生のみ全問主学年）
- 答え合わせ・結果画面・見返し画面・カレンダーはすべて出題数に連動して表示件数が変わる（画面固定表示の「10問」表記は撤廃し、実際の問題数を表示する）。
- バッジの「がんばった回数」「週の習慣」等は**実施した回数（日数）で判定**しており、1回あたりの問題数（10問/20問）による影響は受けない（現状のまま）。

### 4.1.3 出題順序の選択（v2.7 新規・実装完了）
- プロフィールごとに **ランダム／五十音順** を選択できる（§3.5.2）。
- **ランダム**：これまでどおり、選ばれた問題をシャッフルして表示する（既定値・現行動作を維持）。
- **五十音順**：選ばれた問題を、教育漢字配当表の並び順（＝ `kanji-data.js` に収録されている順序。実質的にあいうえお順に近い並び）に沿って表示する。
  - 前日の不正解の復習や学年の重み付け（§4.1.1）で「どの漢字が選ばれるか」は変わらない。**変わるのは表示する並び順のみ。**

### 4.1.4 学習モード：予習／復習（v2.8 新規・実装完了）

**背景・課題**：これまでの出題（§4.1.1）は「プロフィールの学年を中心に、まだ習っていないかもしれない漢字も含めて」出題する、いわば**予習型**の出題ロジックだった。これに加え、**学校で実際に習った範囲だけに絞って復習したい**という要望に対応する。

**仕様**
- プロフィールごとに **予習モード／復習モード** を選択する（§3.5.2）。
- **予習モード**：§4.1.1の学年重み付けロジックをそのまま使用する。**動作は一切変更しない。**
- **復習モード**：出題プールを「習った漢字リスト」（§4.1.5）に限定する。学年重み付け（9:1等）は使わず、**習った漢字リストの中から**、以下の優先順で選ぶ：
  1. **前日に間違えた漢字のうち、習った漢字リストに含まれるもの**（＝「習った漢字」かつ「前回間違えた」の両方を満たす問題）を優先的に出題する。
  2. 残りの枠は、習った漢字リストの中から**正解率の低い漢字・未出題の漢字を優先**して選ぶ（＝新たな漢字の出題）。
  3. 1・2をあわせて出題数（§4.1.2：10問または20問）に達するまで選ぶ。
- 出題数（10問/20問）・出題順序（ランダム/五十音順）の設定は、予習・復習どちらのモードでも共通して適用される。
- 学年の重み付け（9:1）は**予習モードのみの概念**であり、復習モードには存在しない（習った漢字リストという単一のプールから選ぶため）。

### 4.1.5 習った漢字リスト（v2.8 新規・実装完了）

**仕様**
- 「習った漢字リスト」＝ **(A) プロフィールの学年より下の学年の漢字（自動・全て）** ＋ **(B) 現在の学年のうち、ユーザーが明示的に「習った」とチェックした漢字**。
  - 例：プロフィールが5年生の場合 → 2・3・4年の漢字は自動的に全て「習った」扱い。5年の漢字は、ホーム画面の「習った漢字を選ぶ」（§3.5.2）でチェックした分だけが対象になる。
- チェックは学年ごとに保持され、学年を変更してもチェック内容は消えない（§3.5.3）。
- 復習モードで出題プールが空（=現在の学年で1つも選択しておらず、かつ一番下の学年〈2年生〉の場合など）になった場合は、ホーム画面に「まずは『習った漢字を選ぶ』でチェックしてね」という案内を表示し、テスト開始をブロックする。

### 4.1.6 熟語問題の全文字習得判定と代替問題（v2.8.1 バグ修正・実装完了）

**背景（発見された不具合）**：v2.8の復習モードは「出題が登録されている行の代表漢字1文字」だけで習得判定していた。しかし熟語問題は答えが複数文字（例：「複雑」を書かせる問題は「複」の行にあるが答えは複＋雑）。そのため代表漢字（複）が習得済みなら、**答えに含まれる未習の相方漢字（雑）ごと出題**されてしまい、§4.1.5の意図に反していた。該当は熟語問題438問中111問。

**修正仕様**
- 復習モードでは、**答え（`{{ }}`内）の全漢字が習得済み**の場合のみその問題をそのまま出題する。
- 未習の相方漢字を含む場合は、**その漢字単独で成立する代替問題に差し替える**（`sentence-data-alt.js`、71字：訓読み41字＋下位学年の相方を持つ熟語30字）。
  - 例：訓読みあり → 園→「はなの園」／音読みのみ → 複→「複数」（複雑ではなく）。
- 代替も無く未習相方を含む38字（例：科・京・雑・演）は、相方も習得済みになるまで復習出題しない（既知の制約。無理な代替を作らない方針）。
- 代替問題の周囲は誤変換防止のため既習漢字化せず、対象以外はかな表記。
- **予習モード（§4.1.1）は影響を受けない**（従来どおり）。

### 4.2 問題タイプ（v2.1 改訂）
| タイプ | 出題内容（画面表示） | 紙の回答 |
|--------|---------------------|---------|
| 書き（訓読み中心） | かなの短文を提示（下線部が対象） | 下線部を漢字**（送りがなを含む）**で書く（§4.2.2） |
| 熟語（音読み中心） | かなの短文を提示（下線部が対象） | 下線部の熟語を漢字で書く |

- 出題はすべて**5〜10文字程度のかな短文**とする。単語だけの出題はしない。
  - NG：「その」→ 園
  - OK：「うたをうたう」→ 歌を歌う ／「こうえんであそぶ」→ 公園であそぶ
- 短文データは `sentence-data.js` に全漢字ぶん収録（かな文・漢字文・タイプ）。
- 音読み・訓読み・熟語をバランスよく混在させる。
- **（v2.2）問題文中、出題対象（答えさせる箇所）以外の部分は、極力すでに習った漢字に変換して表示する。**
  - 例：「となりのへやにいどう」（全部かな）→「となりの部屋に移動」（対象の「いどう」以外はすでに習った熟語「部屋」で表示）
  - 全文かな表記は避け、実際の文章に近い読みごたえのある問題にする。
  - データ形式を `{tpl, reading, type}` に刷新（`tpl` は `{{対象}}` マーカーで出題箇所を示す既習漢字まじりテンプレート）。
  - **実装方法**：`kanji-data.js` の熟語（2文字以上の単語＋読み）を辞書化し、既存の短文データへ機械的に置換するスクリプト（`scripts/convert_sentences.py`）で全550件超を一括変換。
    - 単漢字の訓読み（1文字の読み）は助詞（の・に・を等）と同音衝突し誤変換を起こすため、**辞書化の対象外**とした（例：「の」が「野」に誤変換される事故を確認し除外）。
    - 変換後、辞書内の熟語どうしの同音異義語衝突（例：「こうじょう」＝向上／工場）を全件突合検証し、誤った語が採用されていないことを確認済み。

### 4.2.1 用語の注記
- 「用例」タイプは v2.2 にて「書き」「熟語」の2タイプに整理・統合済み（4.2参照）。

### 4.2.2 送りがなを含めた下線・正解（v2.9 仕様変更・実装完了）

**目的**：送りがなのある訓読みで、送りがなも含めて学べるようにする。下線を「漢字になる部分」だけでなく**送りがなも含めた範囲**に引き、正解も送りがな込みで書かせる。

- **対象**：送りがなを持つ**訓読み**の「書き」タイプのみ。
- **対象外**：
  - 送りがな無しの訓読み（例：顔＝かお、雲＝くも、羽＝はね）→ 従来どおり漢字部分のみ。
  - 「熟語」タイプ（音読み中心、送りがなの概念がない）→ 従来どおり。
- **変更例**

  | 漢字 | 現状（下線＝漢字部分のみ） | 変更後（下線＝送りがな込み） | 正解 |
  |------|--------------------------|----------------------------|------|
  | 強 | <u>つよ</u>いかぜ | <u>つよい</u>かぜ | 強い |
  | 楽 | <u>たの</u>しい | <u>たのしい</u> | 楽しい |

### 4.2.3 接続助詞をまたぐ複数下線（v2.9 仕様変更・実装完了）

**目的**：1つの短文に出題対象が2つあり接続助詞（を・と等）でつながる場合、対象ごとに独立した下線を引き、助詞は下線対象外（地の文）にする。

- **変更例**

  | 漢字 | 現状（1本の下線） | 変更後（2本の下線・助詞は地の文） | 正解 |
  |------|------------------|-----------------------------------|------|
  | 歌 | <u>うたをうたう</u> | <u>うた</u>を<u>うたう</u> | 歌を歌う |
  | 数 | <u>かずをかぞえる</u> | <u>かず</u>を<u>かぞえる</u> | 数を数える |
  | 親 | <u>おやとこ</u>ども | <u>おや</u>と<u>こ</u>ども | 親と子ども |
  | 組 | あか<u>ぐみとしろぐみ</u> | あか<u>ぐみ</u>と<u>しろぐみ</u> | あか組としろ組 |
  | 伝 | <u>でんごんをつたえる</u> | <u>でんごん</u>を<u>つたえる</u> | 伝言を伝える |
  | 祖 | <u>そふとそぼ</u> | <u>そふ</u>と<u>そぼ</u> | 祖父と祖母 |

- 該当は現データ中**6件**（上記全て）。
- **§4.2.2 との複合**：「伝言を伝える」は§4.2.3（助詞で分割）と§4.2.2（送りがな込み）が同時に適用され、下線は「でんごん」「つたえる」の2本、「を」は地の文、送りがな「える」は下線内・正解は「伝言を伝える」。

### 4.3 出題ロジック（適応的出題）
1. **過去に間違えた問題を、§4.3.2の間隔反復スケジュールに基づき優先**して当日の出題に含める。ただし§4.3.1の上限を超えない範囲とする。
2. 残りは**正解率の低い漢字・未出題の漢字を優先**して選出。
3. 問題タイプ（書き・熟語・用例）をバランスよく混ぜる。
4. 出題順はシャッフルする。

### 4.3.1 過去に間違えた問題の出題比率上限（v2.10 仕様変更・実装完了）

**目的**：復習（間違え直し）の比重が大きくなりすぎて、新しい漢字を学ぶ機会が圧迫されないようにする。

- 1回の出題に占める「過去に間違えた問題（§4.3.2の間隔反復スケジュール対象）」の割合は、出題総数の**30%まで**とする（小数点以下切り捨て）。
  - 例：10問出題 → 過去に間違えた問題は**最大3問**まで
  - 例：20問出題 → 過去に間違えた問題は**最大6問**まで
- 上限を超えて「本日が出題対象日」の間違い問題がある場合は、§4.3.2のスケジュールで**対象日を過ぎている日数が長いもの**を優先し、残りは翌日以降に持ち越す（消えずに繰越し）。
- 予習モード・復習モードの両方に適用する（現状どちらも「前日不正解」を優先する仕組みを持っているため、両方を§4.3.2の間隔反復ベースに置き換える）。

### 4.3.2 間違えた問題の間隔反復（スペースド・リピティション）出題（v2.10 仕様変更・実装完了）

**目的**：一度間違えた漢字を、間隔をあけて繰り返し出題する（間隔反復学習法）ことで、短期的な暗記でなく長期記憶への定着を図る。

- **頻度ステージ**：以下8段階を、間隔の短い順（ステージ0〜7）とする。
  ステージ0＝翌日／1＝3日後／2＝5日後／3＝1週間後／4＝10日後／5＝15日後／6＝20日後／7＝30日後
- **開始**：まだ追跡対象でない漢字（`key`単位、既存`stats`と同じ粒度）を初めて間違えると、**ステージ0（翌日）**から追跡を開始する。次回の出題対象日＝間違えた日＋ステージ0の日数。
  - すでに追跡中の漢字を間違えた場合は、この「開始」ではなく下記「対象日に間違えた場合」（1つ前のステージに戻す）を適用する。
- **対象日に正解した場合**：連続正解数を+1し、
  - **連続正解2回で打ち切り**（追跡対象から外し、通常の正解率ベース出題§4.3の2に戻る）。
  - 2回に満たなければ、ステージを1つ進め（上限はステージ7）、次回対象日＝その正解した日＋新ステージの日数。
- **対象日に間違えた場合**：連続正解数を0にリセットし、**ステージを1つ戻す**（下限はステージ0）。次回対象日＝その間違えた日＋新ステージの日数。
  - 例：ステージ3（1週間後）で間違えた → ステージ2（5日後）に戻り、その日から5日後が次の対象日。
  - ステージ0で間違えた場合はステージ0のまま（翌日が次の対象日）。
- **対象日の扱い**：対象日当日に出題できなかった場合（アプリを開かなかった等）は、次に開いた日に「出題待ち」として繰り越す（消滅させない）。
- **対象範囲**：予習モード・復習モードの両方に適用する（両モードとも同じ間隔反復の追跡データを参照・更新する）。
- **データ保持**：漢字（`key`）ごとに「現在のステージ（0〜7）」「次回対象日」「連続正解数（0〜1）」を保持する。既存の`stats`（tested/correct集計）とは別に、間隔反復専用の状態を新設する。
- **§4.3.1との関係**：ある日に対象日を迎える間違い問題が§4.3.1の上限（30%）を超える場合は、対象日超過日数が長い方を優先し、残りは翌日以降に繰り越す。

### 4.3.3 §4.3.1・4.3.2の実装内容（v2.10）
- プロフィールごとのデータに`srs`（漢字keyごとに`{stage, correctStreak, nextDue}`）を新設。既存の`history`（前日不正解の判定に使用）に代わり、これを唯一の情報源として出題判定する（旧`Store.getYesterdayWrong()`は削除）。
- 正誤の記録時（`Store.recordAnswer`と同じタイミング）に`Store.recordSrsResult(key, correct)`を呼び、ステージ・次回対象日を更新する。
- 出題選定（`Quiz.selectSrsDue(cap)`）は、対象日を迎えた漢字を超過日数の降順で取得し、上限件数までを返す。未習の相方を含む場合は既存の代替問題（`getReviewCandidate`）で救済し、救済できないものは見送り（対象日はそのままのため自然に繰り越される）。
- **実装中に発見・修正した不具合**：日付計算で`Date.toISOString()`を使うとUTCに変換されるため、JST（UTC+9）環境では計算結果が1日ずれるバグがあった。ローカルの年月日を直接組み立てる方式に修正し、ブラウザ実機で検証済み。
  - 同種のバグが既存の`weekKey()`（週の習慣バッジ集計）にも見つかったが、本v2.10の対象外のため別途対応する。

## 5. 採点・記録仕様（v2.0 中核）

### 5.1 答え合わせフロー（v2.1 改訂：一括答え合わせ方式）
```
1. PC画面に第1問〜第10問を連続で出題（答えは見せない）
2. 児童は紙に①〜⑩の番号をつけて漢字を書いていく

4. 児童本人 or 保護者が紙と正解を照合
```

- ⭕❌を10問すべて選ぶまで記録ボタンは押せない（未採点防止）。

|------|------|

### 5.3 テストの見返し（v2.5 新規・実装完了）

**背景・課題**：現状は結果画面（§5.1の手順6）を離れると、その回にどんな問題が出て、どう答えたかを二度と確認できない。「答え合わせをした後、問題を見返したい」という要望に対応する。

**仕様（確定・v2.5改訂：カレンダー経由の見返しを追加）**
- **アクティブなプロフィールの、いつのテストでも見返せるようにする。**（直近2回はホーム画面から即アクセス、それ以前も含む全履歴はカレンダー経由でアクセス）
- **エントリーポイント**
  1. **結果画面**：現在の「まちがえた問題」一覧に加え、**「📖 この回をぜんぶ見返す」ボタン**を新設し、その場で見返し画面（下記）に遷移できるようにする。
  2. **ホーム画面**：新しいカード「📖 テストの見返し」を追加し、アクティブなプロフィールの**直近2回分**（実施日・スコア）をボタン形式で並べる。タップすると見返し画面に遷移する。
     - 例：`[7/4(金) 7/10問正解]` `[7/3(木) 9/10問正解]`
     - 実施回数が2回未満の場合は、ある分だけ表示する（0回なら本カード自体を表示しない）。
  3. **学習カレンダー（v2.5拡張）**：✅の付いた日をタップした際に表示される「その日のスコア」に加えて、**「📖 この日の問題を見返す」ボタン**を追加する。これにより**カレンダーに記録が残っている全期間**のテストを見返せる（直近2回に限らない）。
- **見返し画面（新規）**
  - ヘッダーに対象日・スコア（例：「2026-07-04のテスト　7/10問正解」）を表示。
  - その回の**全問題を出題順に一覧表示**し、各問題に以下を表示する：
    - 問題文（対象箇所を漢字にした完成形の文）
    - 自己採点の結果（⭕ せいかい／❌ まちがい のバッジ）
  - 「🏠 ホームにもどる」ボタンで戻る（カレンダー経由で開いた場合はカレンダーへ戻る）。
- **データ**：既存の学習履歴（`history[日付] = {questions, answers, score}`、§3.5.3）をそのまま利用する。**データ構造の変更は不要**（後方互換）。カレンダーには元々全期間の履歴が保持されているため、見返し画面を日付指定で呼べる形にするだけで全履歴に対応できる。

### 6.2.1 バッジ体系（v2.6 新規・実装完了）

**背景・課題**：現行バッジは「連続日数」中心（3/7/30/100日連続）＋累計解答数（50/200問）の6種のみ。毎日は難しくても週に数回続けられている子や、連続は途切れても長く学習を続けている子を称える仕組みがなかった。**「毎日でなくてもコツコツ続けている」ことを多面的に称える**よう、5カテゴリへ拡張する。

**設計方針**
- **連続（ストリーク）だけに偏らない。** 「連続」「累計回数」「週の習慣」「長期継続」「複数学年」の**5つの観点**で継続を評価する。
- **最初はこまめに、だんだん間隔を広げる。** 学習を始めたばかりの子がすぐ達成感を得られるよう、序盤のバッジは細かく（5回ごと等）配置し、習慣化を後押しする。
- **獲得済みも未獲得も一覧表示（ラベル付きコレクション）。** 未獲得はグレー表示にし「次はこれを目指す」と分かるようにして、継続の動機付けにする。

**バッジ一覧（アイコンは実装時に微調整）**

**A. 連続日数（ストリーク）** — 既存を継承

**D. 長く続けた証（長期継続）** — NEW（「低学年→高学年まで続けている」長い継続を表現）

| 🌙 半年つづけた | 180日以上 かつ 累計60回以上 |
| 🎊 1年つづけた | 365日以上 かつ 累計120回以上 |
- 「経過期間だけ」では長期放置でも達成できてしまうため、**期間と実施回数の両方**を条件にして「実際に続けている」ことを担保する。

- 判定・表示は**アクティブなプロフィールごと**に行う（§3.5）。

**表示（ラベル付きコレクション）**

- バッジ数が増えるため、ホーム内で縦にスクロールして見られる形にする（別画面化はしない）。

## 7. 漢字データ仕様

- 形式：`[漢字, 学年, [音読み], [訓読み], [[熟語, よみ], ...]]`
- 収録：小学2〜5年生の配当漢字（約550字）。
- 用途：出題文の生成、答え合わせ、一覧・モーダル表示。

### 7.1 対象学年の拡張（v2.7 新規・実装完了）

- **小学6年生**：文部科学省の学年別漢字配当表に基づく公式の6年生配当漢字（191字）をそのまま収録する。
- **中学1〜3年生**：中学校の漢字は小学校のような**国の公式な学年別リストが存在しない**（常用漢字のうち教育漢字を除いた約1,110字を3学年で学習する、という大枠のみが定められている）。そのため本アプリでは、**一般的な中学校国語教科書・漢字ドリルで各学年の「新出漢字」として共通して扱われる代表的なものを参考に、中1・中2・中3それぞれ100〜150字程度（合計300〜450字程度）を収録**する。
  - 注記：これは教育委員会・教科書会社の公式な年度配当ではなく、**アプリ独自の目安の区分**である旨を漢字いちらん画面・要件定義書に明記する。
  - **今回はこの300〜450字程度を対象範囲とし、常用漢字の残り（約1,110字）を全て網羅することはしない。** 全常用漢字への拡張は「発展編」として今後の展開（§10.2）に位置づける。
- 学年コードは文字列の配列 `['2','3','4','5','6','J1','J2','J3']` の順で管理し、「一つ下の学年」は算術（学年番号−1）ではなく**この配列上の1つ前の要素**として求める（中学1〜3年という非数値の連なりに対応するため）。
- 出題文データ（`sentence-data.js`）も同形式（`{tpl, reading, type}`）で6年生・中学1〜3年生ぶんを追加する。既存の変換スクリプト（`scripts/convert_sentences.py`）を再利用し、既習漢字への変換と同音異義語の突合検証を行う。
- **画面への影響**：学年タブ（ホーム／しんちょくマップ／漢字いちらん）に「6年」「中1」「中2」「中3」を追加する。

5. 「10問」を前提にしていたUI文言（ホームの開始ボタン・出題中の紙のヒント・結果画面の分母表示等）を**すべて実際の問題数を参照する動的表示**に変更。
6. バッジ「複数学年チャレンジ」の階層条件を、対象学年数の増加（4→8段階）に合わせて更新。

**影響が大きい作業（工数の見積り）**
- 新規漢字データ（6年生191字＋中学1〜3年合計300〜450字、総計約500〜650字）の読み・熟語・出題短文の作成が最大の工数。6年生は公式配当のため対象漢字の選定に迷いはないが、中学1〜3年は§7.1の注記どおり「アプリ独自の目安」である点を踏まえて収録する。

**データ構造への影響**
- `qCount`・`orderMode` はプロフィールへの**追加フィールドのみ**（後方互換、未設定時はデフォルト値にフォールバック）。
- 学年コードの配列管理化は**内部ロジックの一般化**であり、既存の2〜5年データ・保存済み学習履歴に影響しない。
